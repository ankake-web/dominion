### 新機構は4つだけ（うち2つは既存流用）
1. **村人（Villager）＝完全新規**。アクションフェイズ中に1個＝+1アクション。per-player 数値。
2. **財源（Coffers）＝既に実装済み**（ギルド §0-4 の `p.coffers` / `COFFERS_SPEND`）。ルネサンスは供給源が増えるだけ。
3. **アーティファクト（Artifact）＝新規**。**同時に1人しか持てない**能力カード（奪い合い）。非カード＝保存則tally対象外。
4. **プロジェクト（Project）＝新規だが横型基盤を流用**。`DOM.LANDSCAPES`／`BUY_EVENT` 購入ディスパッチャ（§0-20）と同型。差分は「一度買うと永続」「同じものを2回買えない」「1人2つまで」。

### 無いもの（実装が要らない＝重要）
- **夜フェイズ（Night）は1枚も無い**（Nocturne の機構）。新フェイズ追加なし。
- **負債・ポーション・避難所・プラチナ/植民地・リザーブ・トラベラー・混合山・分割山・山トークン（`p.pileTokens`）も無し**。
- 持続（Duration）は **貨物船・研究の2枚だけ**＝既存機構でそのまま扱える。
- 命令（Command）は **王笏（2024エラッタで Command 種別を獲得）** のみ。

### 出典の質
- **high**：王国25枚・プロジェクト20枚・アーティファクト5枚のコスト/種別/現行テキスト／2019・2022・2024 エラッタ。RGG PDF 逐語＋公式Errata＋日本語wiki の3系統一致。
- **medium**：日本語版カードの**一字一句**（ホビージャパン実物カード画像は未確認＝wikiwiki 表記に依拠）。2022年の Coffers/Villagers タイミング改訂の**原文投稿**（Donald X. のフォーラム投稿は bot 遮断で未取得。Complete Rules v11 と Wiki で確認）。
- **⚠️未確定は §6**。

## 1. 新機構（実装者向け）

### 1-1. 村人（Villagers）★完全新規
- カードの「+1 Villager」＝共通のコイントークン置き場からトークンを取り、**財源／村人マットの村人側**に置く。上限なし・ターン跨ぎで持ち越し・VPにならない。
- **使用＝自分のアクションフェイズ中ならいつでも**、好きな個数をまとめて取り除ける。**1個＝+1アクション**。
- **財源との違い**：財源＝ターン中いつでも +$1／村人＝**アクションフェイズ限定**で +1アクション。**同じトークンだが別枠で、混ぜて使えない**。
- **アクション権0でも使える**（アクションフェイズはアクション権0で自動終了しない）。本エンジンは `END_ACTION_PHASE` が明示 action なので相性が良い。
- **購入フェイズに入ったら使えない**（戻れない）。ただし**ヴィラ（帝国）でアクションフェイズに戻れば、そのフェイズで再び使える**。
- **公開情報**（相手も数えられる）＝`maskStateFor` でマスクしない。
- 実装：`p.villagers`（数値・`createInitialState` で 0 初期化・`p.coffers` と同型）＋新 action **`SPEND_VILLAGER`**（アクションフェイズ・任意個数・`t.actions += n`）。`PLAYER_ACTIONS` に登録。UI＝村人バッジ＋「🧑 村人を使う」ボタン（`modalAmount` 流用）。CPU＝`chooseAction` でアクション権が尽きて手札にアクションが残っていれば消費（終端保証＝0個なら送らない）。
- ルネサンスの供給源：劇団(+4)／追従者(獲得時+2)／パトロン(+1)／徴募官(廃棄コスト$1につき+1)／絹商人(獲得/廃棄時 +1財源+1村人)／彫刻家(財宝を獲得したら+1)／プロジェクト 学園・道路網(×)・**学園**（アクション獲得で+1）。

### 1-2. 財源（Coffers）★既存流用（ギルド）
- **ギルドの財源と完全に同一機構**（同じトークン）。`p.coffers`＋`COFFERS_SPEND`（購入フェイズ・1個=+$1）が実装済み（§0-4）。**新規実装は不要**。
- ルネサンスの供給源：ドゥカート金貨(+1)／香辛料(獲得時+2)／パトロン(アクションフェイズ中に公開されたら+1)／絹商人／剣客(+1・4個以上で宝箱)／悪党(+2)／出納官(間接)／プロジェクト ギルド集会所・城門(×)・**ギルド集会所**（財宝獲得で+1）・**野外劇**・**探査**。
- **現行ルール（2022改訂）は「自分のターン中いつでも使える」**。既存実装は「購入フェイズのみ」＝**2022改訂前の挙動**。→ **本実装はそのまま（購入フェイズのみ）で出荷してよい**（§6-1）。ただし**剣客（Swashbuckler）の『財源4個以上』判定**と絡むので §6 を読むこと。

### 1-3. アーティファクト（Artifacts）★新規
- 5種（旗／角笛／鍵／ランタン／宝箱）。**それぞれ1枚しか存在せず、ある時点で持てるのは0人か1人**。「取る(take)」とき、他プレイヤーが持っていれば**その人から奪う**（相手は即座に能力を失う）。
- **「取る」は「獲得(gain)」ではない**＝獲得時トリガー・廃棄時トリガーは一切発火しない。**デッキに属さず、決して in play にならず、VPにも3山終了にも無関係**。
- **公開情報**。取ったターンから機能する（**鍵だけは例外＝ターン開始時効果なので取ったターンには恩恵なし**）。
- **セットアップ**：国境警備隊（→角笛・ランタン）／旗手（→旗）／剣客（→宝箱）／出納官（→鍵）が王国にあるときだけ、対応するアーティファクトをサプライの脇に置く。誰も持っていない状態から始まる。
- 実装：**`state.artifacts = { flag:席番号|null, horn:…, key:…, lantern:…, treasure_chest:… }`**（トップレベル・公開スカラー・**非カード＝invariants の tally に数えない**・`maskStateFor` は clone でそのまま残る＝`state.pileVP`／`state.pileDebt` と同型）。奪取＝所有者を上書きするだけ。
- **cost は持たせない**（`cost:null`）。`DOM.CARDS` に $0 カードとして入れると **3山終了・闇市場デッキ・GAIN_ORDER・汎用「≤$N獲得」に混入して事故る**（§6 の4系統除外チェックリスト＝賞品Prizes で実際に踏んだ罠）。**`DOM.LANDSCAPES` 側に `kind:'artifact'` で置き、購入経路から完全に除外すること**。

### 1-4. プロジェクト（Projects）★新規（横型基盤を流用）
- 20種・コストは**コインのみ**（$3〜$8）。**購入フェイズにカードを買う代わりに買う**＝**購入権を1消費**し、コインを支払う。**カードは獲得しない**。代わりに自分のキューブを1個置き、**以後ゲーム終了まで永続的に自分にその能力が適用される**。
- **キューブは各プレイヤー2個だけ ＝ 1人が持てるプロジェクトは最大2つ**。**同じプロジェクトに自分のキューブを2個置けない ＝ 同じプロジェクトを2回買えない**（イベントとの決定的な違い）。一度置いたキューブは取り除けない。
- **複数のプレイヤーが同じプロジェクトを買える**（各自が自分のキューブを置く）。効果は各自に独立して適用される。
- **プロジェクトを買うのは「カードを買う」ではない**：
  - **コスト軽減（橋／街道／橋の下のトロル／発明家／運河）を一切受けない**。
  - **購入時トリガー（商人ギルド／堀の外の悪党／値切り屋／公会堂 等の「カードを買ったとき」）も発動しない**。
  - **獲得でもない**（獲得時トリガー不発／探査の「カードを獲得しなかった」条件を満たしたまま）。
  - **負債（帝国）が残っている間は買えない**（カード・イベントと同じ）。
- 1ターンに購入権が複数あれば、カード・イベント・プロジェクトを混ぜて買える。
- **横型の採用枚数は合計2枚まで**（プロジェクト＋イベント＋ランドマークの合算）＝**本アプリの既存仕様（0〜2枚・王国山は常に10）と完全一致**。プロジェクトを既存の landscape 抽選枠に混ぜてよい。若き魔女の Bane にはできない。
- 実装：
  - **`state.projects = [id, id]`**（対局中不変・公開・`state.events`／`state.landmarks` と同型）＋**`p.projects = [id…]`**（最大2・公開）。
  - 新 action **`BUY_PROJECT`**（`BUY_EVENT`＝§0-20 の購入ディスパッチャを丸ごと流用）。**`PLAYER_ACTIONS` に登録必須**。
  - **`DOM.engine.canBuyProject(state, pi, id)` を正本にする**（engine拒否・CPU非提案・UIボタン無効の3面が同じ述語を見る＝`canBuyEvent` と同型）。条件＝`state.projects` に含まれる／`coins >= cost`（**コスト軽減を通さない定数コスト**）／`buys >= 1`／`p.debt === 0`／`!p.projects.includes(id)`／`p.projects.length < 2`。
  - **`t.treasuresLocked = true` を立てる**（§0-21 の全体ルール＝一度でも購入したらそのターン財宝を出せない）。**`t.buysMade++` も忘れない**（使者 messenger の「そのターン最初の購入」に数える）。
  - CPU＝`bestEventBuy` の隣に `bestProjectBuy`（**buy可能かつ debt=0 かつ未所持かつ枠が空いているものだけ返す**＝engine が拒否しない＝無限ループ防止）。
  - UI＝盤面のランドスケープ帯にプロジェクトを表示＋**自分のキューブ（●）と相手のキューブ**を出す。

### 1-5. 対局準備（Setup）
1. 王国カードは通常どおり10種。プロジェクトは10種に含まれない**別枠の横型**（プロジェクトが出ても王国10枚が揃うまでランダマイザをめくり続ける）。
2. 横型は合計2枚まで（推奨）。プロジェクトのみなら最大2枚。
3. プロジェクトを使うゲームでは、各プレイヤーが**キューブを2個**取る（＝活性化できるプロジェクトは最大2つ）。
4. 財源または村人に言及するカードが1枚でもあれば、各プレイヤーに財源／村人マットを配り、コイントークンをサプライの近くに置く。
5. **国境警備隊・旗手・剣客・出納官のいずれかが王国にあれば**、対応するアーティファクトをサプライの脇に置く。
6. 開始デッキ・サプライ・終了条件は通常どおり。**プロジェクトもアーティファクトも山数・3山切れに影響しない**。
7. 負債・ポーション・夜フェイズ・避難所・プラチナ/植民地 などの特別セットアップは**無い**。

## 2. 王国カード 25枚

> `cost` はコイン。全カード **debt=0 / potion=0**。`text` は日本語版準拠（wikiwiki）。**裁定はすべて現行（エラッタ後）**。

### $2

#### `border_guard` 国境警備隊（Border Guard）— $2 / アクション
```
+1 アクション
あなたのデッキの上から2枚のカードを公開する。そのうち1枚をあなたの手札に加え、残りを捨て札にする。
両方ともアクションカードの場合、ランタンか角笛を受け取る。
```
- **アーティファクト2種を使う**：**角笛**「各ターンに1度、あなたが場から国境警備隊1枚を捨て札にするとき、あなたのデッキの上に置いてもよい」／**ランタン**「あなたが**使用する**国境警備隊は、カードを3枚公開し2枚を捨て札にする（角笛を取るには3枚すべてがアクションである必要がある）」。
- **ランタン非所持**：上2枚公開→1枚を手札・1枚を捨て札。両方アクションなら**「ランタンか角笛」を取る＝強制の二択**（既に持っている方を選んで実質 no-op も可）。
- **ランタン所持**：上3枚公開→1枚を手札・2枚を捨て札。**3枚すべてアクションのときだけ角笛を取っても“よい”＝任意**。
- **公開枚数が足りない**（非ランタンで2枚未満／ランタンで3枚未満）ならアーティファクトを取らない。1枚しか公開できなければその1枚を手札へ。
- **相続（Inheritance）の屋敷**：ランタンは効く（Donald X. 裁定＝ランタンは「あなたが使用する国境警備隊」の on-play 自体を書き換える）が、**角笛では山札の上に置けない**（場から捨てるのは名前が「屋敷」のカードだから）。
- **命令（大君主/はみだし者）や船長/王子でプレイした国境警備隊にもランタンは適用される**（2019エラッタの主眼＝下記）。
- **パトロン**を国境警備隊で公開すると +1財源（2枚ともパトロンなら +2財源）。
- **実装メモ**：`applyEffect` の border_guard 内で `state.artifacts.lantern === pi` を見て公開枚数を 2/3 に切り替える。角笛は **`cleanupAndAdvance` の「場を捨てる」処理**にフック（**先引きより前**！ §5-1）。`t.hornUsed` で1ターン1回。

#### `ducat` ドゥカート金貨（Ducat）— $2 / 財宝（coin:0）
```
+1 財源
+1 カードを購入
---
このカードを獲得するとき、あなたの手札から銅貨1枚を廃棄してもよい。
```
- **コインは出ない（coin=0）**。使用時は +1財源 と +1購入 のみ（RGG逐語 "you get no $"）。
- 獲得時の銅貨廃棄は**任意**・**手札の銅貨のみ**（山札・捨て札・場は不可）。手札に銅貨が無ければ**pending を立てない**。
- **購入以外の獲得でも発動**（工房・改築・闇市場・**出納官で廃棄置き場から獲得**した場合も）。**相手のターンの獲得でも本人が廃棄できる**。
- 購入フェイズに買った場合、既に場に出した銅貨は「手札」ではないので廃棄できない。
- **デッキ圧縮にはならない**（銅貨1枚をドゥカート1枚に置換するだけ）＝CPUの購入評価で「圧縮札」と誤評価しないこと。
- **冠/ティアラ/偽造通貨で2回使用すると +2財源・+2購入**＝`applyTreasureEffect` に書く（§0-15）。

#### `lackeys` 追従者（Lackeys）— $2 / アクション
```
+2 カードを引く
---
このカードを獲得したとき、+2 村人。
```
- 獲得時の +2村人は**強制**。購入以外の獲得でも発動（闇市場・廃棄置き場からの獲得を含む）。**廃棄では発動しない**（旗手・絹商人と違う点）。
- 購入フェイズに獲得した村人は、そのターンには使えない（アクションフェイズに戻る手段が無い限り）。未使用分は持ち越す。

### $3

#### `acting_troupe` 劇団（Acting Troupe）— $3 / アクション
```
+4 村人
このカードを廃棄する。
```
- **廃棄は +4村人 の条件ではない**。廃棄に失敗しても村人4個は得る（RGG逐語）。玉座の間で2回使えば**計8村人**（廃棄は1回）。
- **命令（大君主/はみだし者）でサプライの劇団を使用すると廃棄には失敗するが村人4個は得る**（RGG Dark Ages 2022 が名指しで例示＝本プロジェクトの `takeSelf`/`playedByCommand` §0-17 と一致）。
- 実装：自己廃棄札なので **`if (removeOne(p.inPlay,'acting_troupe')) trashCard(...)`** で包む（`takeSelf` を使う）。

#### `cargo_ship` 貨物船（Cargo Ship）— $3 / アクション・持続
```
+2 コイン
このターン中1回、あなたがカード1枚を獲得したとき、そのカードを表向きにして脇（このカードの上）に置いてもよい。
あなたの次のターンの開始時、そのカードをあなたの手札に加える。
```
- 脇に置くのは**任意**。**そのターンの任意の獲得1回**で使える（最初の獲得である必要はない＝複数枚獲得してから決めてよい）。
- **1枚も脇に置かなければ、そのターンのクリンナップで捨て札になる**（持続として場に残らない）。置いた場合のみ次ターンまで場に残る。
- 玉座/行進で複数回使用すると、使用回数ぶん「このターン1回」の能力が独立して得られる。
- 獲得したカードの獲得時能力・物見やぐら・技術革新と**同時誘発**＝**獲得したプレイヤーが解決順を選ぶ**。

- **獲得先が置換されているカード**（守護者＝手札に獲得／遊牧民の野営地＝山札の上に獲得）は、その場所から脇に置ける（獲得後の移動ではないので lose track にならない）。
- 脇のカードは「場に出ている」扱いではない。貨物船が次ターン開始前に場を離れる（増築・行進で廃棄）と持続効果は失われ、**脇のカードは手札に戻らず脇に残り続ける**（得点計算には数える）。
- 実装：新ゾーン `p.cargo = [{ id, cards:[…] }]`（**表向き＝公開**・`allCards`／invariants tally に加算）。`t.cargoShipCharges` でこのターン残り何回置けるか管理。`DURATION_RESOLVERS.cargo_ship` で次手番開始時に手札へ。

- **先に2枚引き、その後で廃棄**（引いたばかりのカードも廃棄対象）。廃棄は**強制**（RGG逐語）。手札0枚なら何も廃棄せず村人も得ない（pending を立てず終端）。
- **コイン成分のみ**：技術者（$0+負債8）→ +0村人／薬剤師（$2+ポーション）→ +2村人。銀貨→ +3村人。
- コスト軽減後の**現在コスト**で数える。$0未満にならない＝最低 +0村人。
- **【重要】行商人（Peddler）のコスト軽減は購入フェイズ限定**＝アクションフェイズにプレイする徴募官で廃棄すると **$8扱い＝+8村人**（有名コンボ）。`cardCost` がフェイズを正しく見ているか確認すること。
- 城塞を廃棄しても廃棄自体は成立＝**+4村人**（城塞は手札に戻る＝`trashCard` の戻り値 false でも村人は入る）。

#### `scepter` 王笏（Scepter）— $5 / **財宝・命令**（coin:0・可変）
```
次のうち1つを選ぶ：
「+2 コイン」
「このターンにあなたが使用し、場に出たままの、命令ではないアクションカード1枚を、再度使用する」
```
- **2024エラッタで種別に「命令（Command）」が追加され、再使用の対象から命令カードが除外された**（日の出づる国の預言「啓蒙」との無限ループ回避）。**日本語版の実物カードは種別「財宝」のみ＝エラッタ前**。実装は**現行（財宝-命令／非命令のみ再演）**。
- 財宝＝**購入フェイズに使用**（アクション権を消費しない）。再使用で +アクションを得ても、購入フェイズなのでアクションカードは使えない（王笏の再使用だけは行える）。
- **前のターンに使用した持続カードは再使用できない**。**同じターンに使用した持続なら再使用でき、その場合 王笏はその持続カードが場を離れるまで場に残る**（玉座×持続と同じ追跡＝`armDuration`）。
- 再使用でカードを引き、その中に財宝があれば**その財宝はまだ使用できる**（`t.treasuresLocked` は「購入」で立つので王笏の再使用自体は無関係）。
- **「場に出たまま(still in play)」が条件**：島（島マット）／宝の地図・祝宴（自己廃棄）／リザーブ（酒場マット）／陣地（脇置き）は再使用できない。
- **命令カードは対象外**：大君主／はみだし者／船長／王子／**相続で命令になった屋敷（`inheritedEstate`＝実装済み）**。→ 王笏の対象述語に命令除外を必ず入れる。
- 「1つを選ぶ」＝**遂行できない選択肢も選べる**（再使用できるアクションが無くても第2案を選べて何も起きない）＝engine は拒否しない。UIは「+2コイン」を勧める。
- 冠/ティアラ/偽造通貨で2回使用する場合、**各回で別々の選択をしてよい** → **`applyTreasureEffect`＋`treasure_replay` ラベル（§0-15）が必須**（王笏は pending を立てる財宝）。
- 王笏自身は財宝なので王笏で王笏を再使用することはできない。

#### `scholar` 学者（Scholar）— $5 / アクション
```
あなたの手札をすべて捨て札にする。
+7 カードを引く。
```
- **先に手札を全部捨て、その後に7枚引く**（RGG逐語）。**引く途中でシャッフルが必要になれば、いま捨てたカードもシャッフルに混ざる**。
- 手札0枚でも7枚引く（学者自身は場にあるので捨てない）。
- 手札を捨てるのは1回の同時捨て札＝`triggerOnDiscard`（トンネル等）は捨てた枚数ぶん発火。
- デッキ＋捨て札が7枚未満なら引けるだけ（合法）。-1カードトークンで7→6。
- ターミナル。玉座で2回使うと1回目の7枚を全部捨てて改めて7枚引く。

#### `sculptor` 彫刻家（Sculptor）— $5 / アクション
```
コスト4以下のカード1枚を獲得し、あなたの手札に加える。
そのカードが財宝カードの場合、+1村人。
```
- 獲得は**強制**かつ**手札に**（RGG逐語 "The card is gained to your hand; that is not optional."）。
- **獲得先の上書き**：遊牧民の野営地（獲得時に山札の上）を彫刻家で獲得しても**手札**に入る（RGG明記）。実装は「獲得先ゾーン=hand」をカード側 on-gain のデッキ置きより優先。
- 「$4以下」＝コイン成分のみ（`costIsPlainCoin`）。コスト軽減後の現在コストで判定（軽減が無ければ **香辛料$5 は取れない**）。
- +1村人は**獲得したカードが財宝の場合のみ**（銅貨・銀貨でも財宝なので +1村人）。
- 獲得したカードの獲得時能力は誘発する（例：**ドゥカート金貨（$2・財宝）**を獲得 → 手札に入り +1村人 ＋ ドゥカートの「手札の銅貨1枚を任意廃棄」も誘発）。
- **$4以下の山が全て空なら何も獲得せず村人も得ない**（終端保証）。
- 彫刻家はアクションフェイズにプレイするので、手札に来た財宝はそのターン使え、手札に来たアクションはアクション権（村人）があれば使える。

#### `seer` 先見者（Seer）— $5 / アクション
```
+1 カードを引く
+1 アクション
あなたのデッキの上から3枚を公開する。その中のコスト2から4のカードをあなたの手札に加える。残りのカードを好きな順番でデッキの上に戻す。
```
- **先に+1カードを引き、その後で（新しい）山札の上3枚を公開**（引いた1枚は公開対象に含まれない）。
- **ポーション費用・負債コストを含むカードは「$2〜$4」に該当しない**（RGG逐語）。

#### `spices` 香辛料（Spices）— $5 / 財宝（coin:2）
```
+2 コイン
+1 カードを購入
---
このカードを獲得するとき、+2 財源。
```
- 獲得時 +2財源は**強制**。**あらゆる獲得経路**で発動（購入・工房系・闇市場・**出納官で廃棄置き場から獲得**）。RGG の Treasurer 項が名指しで例示。
- **相手のターン中の獲得でも発動**（使者・密輸人・複製 等）。物見やぐら/王の直印で捨て札以外へ置かれる獲得でも発動。

#### `swashbuckler` 剣客（Swashbuckler）— $5 / アクション
```
+3 カードを引く
あなたの捨て札置き場にカードがある場合、+1財源。その後、あなたが財源トークンを4個以上持っている場合、宝箱を取る。
```
- **処理順が最重要**：まず3枚引く → **その後で**捨て札が空でないかを判定する。**3枚引く途中でシャッフルが起きて捨て札が空になったら、+1財源も宝箱も得られない**（RGG逐語）。**+3カードは無条件**。
- 条件は入れ子：捨て札が空なら財源も宝箱も一切得られない（財源が4個以上あっても）。
- **4個以上の判定は +1財源を得た後**（プレイ前3個 → +1で4個 → 宝箱を取る）。
- **宝箱（Treasure Chest）＝「あなたの購入フェイズの開始時、金貨1枚を獲得する」**。取るのも金貨獲得も**強制**。剣客はアクションフェイズにプレイするので**取ったそのターンの購入フェイズから**金貨を得る。
- アタックではない。

#### `treasurer` 出納官（Treasurer）— $5 / アクション
```
+3 コイン
次のうち1つを選ぶ：
「あなたの手札から財宝カード1枚を廃棄する」
「廃棄置き場から財宝カード1枚を獲得し、あなたの手札に加える」
「鍵を取る」
```
- +3コインは常に得る。その後、3択のうち1つを**必ず**選ぶ。
- **遂行できない選択肢も選べる**（手札に財宝が無くても「廃棄」を選べる／廃棄置き場に財宝が無くても「獲得」を選べる／**既に鍵を持っていても「鍵を取る」を選べる**＝RGG逐語）。**engine は3択をいつでも受理し、実行不能なら効果なしで pending を閉じる**（拒否すると人間が詰む／CPUが無限ループ）。
- **廃棄置き場からの獲得は「手札へ」直接**（捨て札を経由しない）。これは通常の**獲得**なので獲得時能力が誘発する（RGG逐語：**ドゥカート金貨＝手札の銅貨1枚を任意廃棄／香辛料＝+2財源**）。サプライ山は減らない（3山終了に影響しない）。物見やぐら/交易商人などの獲得時リアクションも適用される。
- **鍵（Key）＝「あなたのターンの開始時、+$1」**。取ったターンには恩恵がない（開始時は既に過ぎている）。**例外＝ピアッツァ（プロジェクト）でターン開始時に出納官がプレイされ鍵を取った場合はその開始時トリガーとして +$1 を得る**。
- 手札からの廃棄は財宝1枚（廃棄時能力は通常どおり発火＝`trashCard` を通す）。
- **【重要】「財宝」の判定は動的**：プロジェクト**資本主義**があると、自分のターン中は「+$を持つアクション」も財宝になる → **出納官で廃棄置き場から増築を獲得できる**（RGG逐語）。`DOM.isType` の静的判定で書くと壊れる（§6-5）。
- 保存則：`state.trash` から `removeOne` して `p.hand` へ（移動元/移動先に注意）。

#### `villain` 悪党（Villain）— $5 / アクション・アタック
```
+2 財源
手札が5枚以上の他のプレイヤー全員は、手札からコスト$2以上のカード1枚を捨て札にする。（できない場合、手札を公開する。）
```
- 対象は**手札5枚以上**の他プレイヤーのみ（4枚以下は何もしない・公開も不要）。判定は各プレイヤーの**解決時点**（馬商人等のリアクションで手札が5枚未満になれば対象外）。
- コスト$2以上のカードがあれば**必ず1枚捨てる**（どれを捨てるかは被害者が選ぶ）。
- **「$2以上」は解決時点の現在コストで判定**。RGG の Canal 項に逐語例：運河があると屋敷は$1なので捨てられない（＝実質「元コスト$3以上」）。
- コスト$2以上が無ければ**手札を公開する**（"reveal" を使う枝）。
- **パトロン相互作用①**：「捨て札にする」は reveal ではない＝**悪党でパトロンを捨てさせられても +1財源は得られない**（RGG逐語）。
- **パトロン相互作用②**：ただし**「手札を公開する」枝は reveal**＝その手札にパトロンがあれば +1財源。通常はパトロン$4なので公開枝に入らないが、**運河（-$1）＋街道/橋 等でパトロンが$1以下になれば成立し得る**＝公開枝の reveal トリガーを省略しないこと。
- 捨て札は通常の捨て札＝トンネル等の `triggerOnDiscard` は発火。
- 堀・灯台等で無効化。**全員が免疫/手札4枚以下でも +2財源は必ず得る**。
- ポーション/負債持ちは**コイン成分が$2以上なら該当**。

## 5. 実装で事故る裁定・エラッタ一覧

### 5-1. 本エンジン固有の罠（最優先）
1. **【角笛】cleanup の先引きより前に処理する**。本エンジンの `cleanupAndAdvance` は「自分の手番終了時に次の手札を先引き」する。角笛の「場の国境警備隊を山札の上に置く」を先引きの後にやると、そのカードは次の次のターンまで引かれない＝角笛がほぼ無効化される。**正しい順序：増築（クリンナップ開始時の廃棄→獲得）→ 角笛（場から捨てるときの置換）→ 場/手札を捨てる → 旗（+1）を乗せて先引き**。
2. **【旗】先引きの瞬間の保持者に +1**。その後に旗が奪われても引いた6枚は返さない。
3. **【ピアッツァ】ターン開始時は `turn.phase === 'action'`**。ドミニオンに「ターン開始フェイズ」は無い。**帝国の冠（crown）は `turn.phase` でモードを決める**ので、ここを 'buy' や 'none' にすると冠が壊れる。
4. **【BUY_PROJECT】`t.treasuresLocked = true` と `t.buysMade++` を忘れない**（§0-21 の全体ルール＝購入したらそのターン財宝を出せない／使者の「最初の購入」）。
5. **【王笏】財宝の2回使用は `applyTreasureEffect` + `treasure_replay`**（§0-15）。王笏は pending を立てる財宝なので「コインだけ足す」実装だと選択が消える。
6. **【アーティファクトを `DOM.CARDS` に入れない】**。$0カードとして入れると 3山終了・闇市場デッキ・GAIN_ORDER・汎用「≤$N獲得」に混入する（§6 の4系統除外チェックリスト）。
7. **【新pendingは4点セット】**（engine reducer＋`PLAYER_ACTIONS`＋CPU `decidePending`＋UI `viewPendingModal`）＋**終端保証**。ルネサンスで新設が要る pending：国境警備隊（公開→1枚選択＋アーティファクト二択）／ドゥカート on-gain／貨物船（脇置きの是非）／増築（クリンナップ）／根城（廃棄）／発明家（獲得）／山村（捨て札から1枚）／司祭（廃棄）／研究（廃棄）／老魔女（呪い廃棄・被害者ごと）／徴募官（廃棄）／王笏（二択＋再演対象）／彫刻家（手札に獲得）／先見者（戻す順序）／出納官（3択＋各枝）／悪党（捨てる/公開・被害者ごと）／大聖堂／城門／野外劇／下水道／星図／サイロ／悪巧み／輪作／技術革新／ピアッツァ（自動）／`SPEND_VILLAGER`／`BUY_PROJECT`。
8. **【必須獲得×候補ゼロ】**（発明家・彫刻家・増築）は「候補ゼロなら獲得せず解決」で終端保証。engine拒否・CPU非提案・UIスキップを**同じ述語**で。

### 5-3. 「reveal」「discard」「trash」の主体・語の使い分け（パトロン/司祭/絹商人/旗手が絡む）
- **パトロン**＝「公開(reveal)」の語を使う効果のみ。捨て札・獲得・廃棄・プレイは公開ではない。**悪党の「手札を公開する」枝は reveal**。
- **司祭／下水道**＝「**あなたが**廃棄したとき」。アタック（詐欺師/騎士/盗賊/山賊/破壊工作員/私掠船）は**被害者が廃棄者**なので、攻撃側には乗らない。
- **旗手／絹商人**＝「**廃棄を実行したプレイヤー**」が旗／トークンを取る。**アタックでは被害者が取る（利敵）**。**待ち伏せのサプライ廃棄では待ち伏せの使用者が取る**（＝`trashCard` を通さないサプライ廃棄経路にもフックが要る）。

   - **本プロジェクトの判断（要ユーザー確認）**：既存の財源は「購入フェイズのみ」＝改訂前。**そのまま出荷しても実害は小さい**（アクションフェイズで$を使う場面は稀）。ただし **語り部（冒険）／闇市場のようにアクションフェイズ中に$を参照する札**があると差が出る。**村人は改訂の有無に関係なくアクションフェイズ限定**なので、村人は現行どおりで問題ない。
2. **⚠️ 剣客（Swashbuckler）× 財源のタイミング**：現行ルール（ターン中いつでも）だと、**剣客の解決中（+1財源の直後・宝箱判定の直前）に財源を使って3個以下に減らし、宝箱の取得を回避できる**と英語Wikiは書く（カード解決への割り込みが公式に認められるかは一次資料で確定できず）。**既存実装（購入フェイズのみ消費）ならそもそも到達しない＝安全側**。意図的簡略化として記録する。
3. **⚠️ 財源／村人の「公開情報」性**：ルールブックに明文の一文が無い（トークンは私的情報ではない、という一般則とオンライン実装の挙動から公開で確実）。**実装は相手席もマスクしない**方針でよい（confidence: high だが逐語ソースは間接）。
4. **⚠️ 山村（Mountain Village）の文言削除の帰属**：現行 Wiki／オンラインは短縮文（`Put a card from your discard pile into your hand;…`）だが、**RGG が現在ホストする PDF（2021刷）は今も `Look through your discard pile and…`**。「2022エラッタで削除された」という説は 2022 Errata 一覧に Mountain Village が無く**裏取りできなかった**。**両文言で挙動は完全に同一**なので実装に影響しない。
5. **⚠️ 資本主義（Capitalism）の実装難度＝要設計判断**：「テキストに +$ を含むアクションは自分のターン中 財宝でもある」＝**カードの種別を動的に書き換える**。本エンジンの `DOM.isType` は静的なので大改修になる。
   - **判定表は静的に作れる**（`DOM.CARDS` の text を機械判定＋目視で「+$ を含むアクション」の集合を確定）。しかし**「財宝として購入フェイズにプレイ→効果を全解決（アタック発動・持続の場残り）」**の経路が要る。
   - **選択肢**：(a) 資本主義を実装する（重い）／(b) **プロジェクト20種のうち資本主義だけ除外して19種で出荷**（`DOM.PROJECTS_RENAISSANCE` から外す）→ **後追いで追加**。**着手前にユーザーに諮ること**。
6. **⚠️ パトロンの「公開」フックの網羅範囲＝要設計判断**：`reveal` の語を使う効果は全拡張に散在する（伝令官・岐路・パトロール・神託・密偵・海賊船・巨人・国境警備隊・先見者…）。全部に共通ヘルパ `revealCards()` を通すのが忠実だが**改修が広い**。
   - **簡略化案**：ルネサンスの公開元（国境警備隊・先見者）＋既存の主要な公開アタック（密偵/神託/大衆/海賊船/巨人）に限定して配線し、**それ以外の公開では発火しない**を「許容簡略化」として記録する（保存則・非ループには影響しない＝過小適用のみ）。**着手前にユーザーに諮ること**。
7. **⚠️ 大聖堂（Cathedral）でデッキが尽きる場合**：全員が大聖堂を買い、圧縮が進みすぎて手札もデッキも空になる懸念が語られるが、**公式裁定は確認できなかった**（0コストのカードは購入できるので通常は進行する）。**既存の終局安全網（150手番ガード）で保護**すれば足りる。
8. **⚠️ 支配（Possession）× 旗／アーティファクト**：被支配者が旗手を獲得/廃棄したとき誰が旗を取るかの公式裁定は取れず（獲得は支配者が代わりに獲得する規則から支配者、廃棄は実際に廃棄した被支配者、と読むのが自然だが未確認）。**ルネサンスと錬金術は出荷セットで同居しない**ので到達不能（§6 の既存方針と同じ）。
9. **⚠️ 増築（Improve）× クリンナップ中の獲得リアクション**：手札を捨てる前なので物見やぐら等で反応できると解したが、一次資料での明示裁定は未確認（実装は「通常の獲得と同じ扱い」で安全側）。
10. **⚠️ 日本語カードの一字一句**：ホビージャパン実物カードの写植は未確認（すべて wikiwiki 表記に依拠）。**効果内容は確定**。加えて**日本語版は2020年2月発売＝パトロン/技術革新/探査/山砦/王笏 は旧テキストで印刷されている**ため、**本アプリの表示テキストは現行（エラッタ後）を採用する**（自作 webp なので矛盾しない）。
11. **⚠️ 王笏（Scepter）の 2024エラッタ後の印刷カード**：現行英文は Dominion Strategy Wiki（Miraheze）と日本語wikiの2系統で一致を確認したが、**RGG の現行刷カード画像そのものは未確認**（RGG がホストするルールブックPDFは2021刷＝エラッタ前）。実装は現行（財宝-命令）で確定。
12. **⚠️ 先見者（Seer）の「戻す順序」の秘匿**：公式ルールブックに明示の一文が無い（Apothecary/Scrying Pool 系の慣行とオンライン実装に準拠して「順序は非公開」と判断）。**非公開側に倒すのが安全**（順序公開は情報漏洩になるが、非公開は不利益が無い）。
13. **⚠️ 山砦（Citadel）× 玉座の間**：「玉座の間が最初のアクションなら玉座の間自身が再演される」は一般ルールからの演繹で、逐語の公式裁定は未確認。追加ターンでの再発動も同様。

## 7. 推奨バッチ計画（安全順）

- **R1＝共通基盤**：`p.villagers`＋`SPEND_VILLAGER`／`state.artifacts`（5種）／`state.projects`＋`p.projects`＋**`BUY_PROJECT`**＋`canBuyProject`／`DOM.LANDSCAPES` に project/artifact を追加（`DOM.CARDS` には入れない）／`build-landscape.js` に project スキン。**この時点では効果ゼロ＝テスト緑を維持**。
- **R2＝王国の素直な15枚**：追従者・劇団・実験・根城・発明家・山村・司祭・絹商人・学者・徴募官・彫刻家・先見者・香辛料・悪党・老魔女（新pendingは4点セット）。
- **R3＝財源/村人・アーティファクトが絡む6枚**：ドゥカート・国境警備隊（角笛/ランタン）・旗手（旗）・剣客（宝箱）・出納官（鍵）・パトロン（**reveal フック＝§6-6 の設計判断**）。
- **R4＝持続・クリンナップ・再演の4枚**：貨物船・研究・増築（**cleanup 先引き問題＝§5-1**）・王笏（`applyTreasureEffect`＋命令除外）。
- **R5＝プロジェクト（軽い順）**：縁日・兵舎・学園・ギルド集会所・道路網・大聖堂・城門・サイロ・悪巧み・輪作・野外劇・下水道・星図・探査・艦隊・ピアッツァ・技術革新・運河・山砦（**資本主義は §6-5 の判断次第で除外**）。
- **R6＝CARD_SET 昇格**：`DOM.KINGDOM_RENAISSANCE` 固定10種＋`renaissance`（王国固定10＋プロジェクト2抽選）＋`random-renaissance`。CPU `bestProjectBuy`。`sw.js` VERSION を上げる。
- 各バッチ末で `node test/invariants.test.js` 緑 → `npm test` 全緑 → 多エージェント敵対レビュー（ルール忠実性/保存則/CPU非ループ/既存機能への退行）。**push は都度ユーザー確認**。
